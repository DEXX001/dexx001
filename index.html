Suite DEXPACK — modules

DXP-SnifferV2
traffic intel

Sniffer réseau en C basé sur AF_PACKET. Capture les paquets bruts sur l’interface (Wi-Fi / Ethernet), avec analyse par protocole (IP / TCP / UDP, etc.). Conçu pour être utilisé après un MITM sur réseau réel.

Brut, sans dépendances, pensé pour apprendre comment les paquets circulent vraiment.
DXP-MITM (ARP Spoof)
network access

Module d’empoisonnement ARP : détournement du trafic entre victime et passerelle. Sert de pivot pour observer, logguer et exploiter le trafic clair (HTTP, DNS, credentials non chiffrés, etc.).

Étape clé pour lier le code bas niveau à des attaques réseau concrètes.
DXP-Scan
recon

Scanner de ports minimaliste en C : TCP 0–1024 avec timeouts personnalisés. Suffisant pour mapper rapidement une cible, sans la lourdeur d’outils externes.

Travail sur la gestion des sockets, des erreurs et des temps de réponse.
DXP-Backdoor (PoC)
access

Implant réseau très simple : reverse shell en C pour comprendre le principe d’un accès persistant, des connexions sortantes et du contrôle distant.

Base pédagogique pour du développement d’implants plus avancés.
